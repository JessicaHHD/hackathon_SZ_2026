# PRD · 故事魔法师 StoryBook MVP

**版本：** v1.0  
**日期：** 2026-04-25  
**状态：** 黑客松 MVP

---

## 1. 产品概述

### 1.1 背景

家长每天陪伴孩子时，常常需要讲故事。现有绘本内容固定，无法融入孩子当天的真实经历，缺乏个性化和情感共鸣。

### 1.2 产品定位

以孩子当天发生的真实事件为输入，AI 自动生成专属绘本故事和插图，让每个孩子都能成为故事的主角。

### 1.3 目标用户

- **主要：** 3-8 岁儿童的家长（睡前故事场景）
- **次要：** 幼儿园教师（教学辅助场景）

## 2. 核心用户故事

| # | 用户故事 | 优先级 |
|---|---|---|
| U1 | 作为家长，我想输入今天发生的事，生成一本专属绘本 | P0 |
| U2 | 作为家长，我想选择不同画风，匹配孩子的喜好 | P0 |

| U5 | 作为家长，我想创建孩子的专属角色，让主角更像自己的孩子 | P1 |

## 3. 功能需求

### 3.1 故事生成（P0）

- 输入：事件描述（文本）、孩子姓名、年龄、画风、页数（3/5/7）
- 快捷主题：勇敢尝试 / 快乐洗澡 / 梦幻冒险 / 学会分享
- AI 生成结构化故事 JSON，包含每页文字、图片描述、中间页分支选项
- 生成时展示 Lottie 加载动画

### 3.2 插图生成（P0）

- 后台并行生成所有页面插图（最多 5 并发）
- 前端每 5s 轮询，逐页更新图片（骨架屏占位）
- 失败兜底：最多重试 3 次，仍失败则显示占位图
- 图片保存到本地 `frontend/generated/`

### 3.3 绘本阅读器（P0）

- turn.js 翻页动画（800ms，双页展开）
- 封面：书皮素材 + 动物角色眼睛跟随鼠标
- 中间页：分支选项按钮，选择后显示剧情回应气泡（3.5s 自动消失）
- 页码指示器 + 上/下页导航
- 生成完成时触发彩纸动画

### 3.4 角色库（P1）

- 创建角色：名字 + emoji 头像
- 选择角色后自动填入故事生成参数
- 持久化到 localStorage

### 3.5 历史记录（P1）

- 已生成绘本列表，显示标题和生成时间
- 点击可重新进入阅读器
- 持久化到 localStorage

---

## 4. 非功能需求

| 指标 | 目标 |
|---|---|
| 故事文本生成延迟 | < 10s |
| 首张插图可见时间 | < 30s |
| 全部插图完成时间 | < 3min（5页） |
| 图片生成失败率 | 有兜底，用户无感知 |
| 前端兼容性 | Chrome / Edge 最新版 |

## 5. 技术架构

```
用户浏览器
    │  Vue 3 SPA（无构建）
    │  turn.js 翻页 · dotLottie 动画
    │
    ▼ HTTP
FastAPI 后端（port 8001）
    ├── POST /api/generate   → 调用 GPT-5.4 生成故事 JSON
    │                          → 启动后台线程并行生成图片
    ├── GET  /api/images/:id → 轮询图片进度
    └── 静态文件服务（frontend/）

后台线程池（max_workers=5）
    └── 调用 gpt-image-2 → 下载图片 → 存 frontend/generated/
```

## 8. MVP 范围与限制

**MVP 包含：**
- 完整生成 → 阅读流程
- 4 种画风
- 角色库 + 历史记录（localStorage）
- 图片失败兜底

**MVP 不包含：**
- 用户账号系统
- 云端存储
- 绘本导出（PDF/分享链接）
- 移动端适配
- 多语言支持
